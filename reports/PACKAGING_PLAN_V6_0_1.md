# 电化学数据处理软件 V6.0.1 打包方案建议

## 目标

在不重写现有前端与后端架构的前提下，将当前 `v6` 工作台整理为可交付、可安装、可独立运行的桌面程序。

## 现状

当前系统本质为：

1. Python 本地服务
- `python run_v6.py server --port 8010`

2. 浏览器前端
- 打开 `http://127.0.0.1:8010/ui`

3. 本地数据文件
- 项目、历史、模板、会话、报告等均保存在本地

这意味着最适合的发布路线不是重写 GUI，而是“保留本地 HTTP 服务，外层加桌面壳”。

## 推荐路线

### 方案 A：桌面壳 + 本地服务

推荐度：高

做法：

1. 启动桌面程序时自动拉起本地 Python 服务
2. 内嵌一个 WebView 打开 `http://127.0.0.1:8010/ui`
3. 关闭桌面程序时同步关闭本地服务

优点：

- 保留当前 UI 与 API 架构
- 改动最小
- 发布成本最低
- 后续前端迭代不需要重写桌面层

可选技术：

- `pywebview`
- `PySide6 + QWebEngineView`

建议优先：

- `pywebview`
  - 接入快
  - 适合当前阶段

### 方案 B：纯浏览器分发

推荐度：中

做法：

1. 提供启动脚本
2. 用户本机启动服务
3. 自动打开默认浏览器

优点：

- 几乎无需额外桌面代码

缺点：

- 产品感较弱
- 用户会感知“这是个本地网页”

适合：

- 内部测试
- 快速验证

### 方案 C：重写为原生 GUI

推荐度：低

原因：

- 当前已有完整前端
- 重写成本高
- 容易引入回归
- 与当前架构方向相反

结论：

- 不建议

## 发布目录建议

推荐将程序与用户数据分离。

### 安装目录

只放程序本体：

- 可执行文件
- 前端静态资源
- 运行依赖
- README
- 更新日志

### 用户数据目录

建议放在：

```text
%USERPROFILE%\\.electrochem\\
```

或按平台规范放到用户应用数据目录。

建议包含：

- `projects.json`
- `processing_history.json`
- `conversation_history.json`
- `process_templates.json`
- `latest_quality_report.json`
- `project_reports/`
- `compare_plots/`
- `llm_config.json`

## 打包前准备

1. 统一路径策略
- 以用户目录为默认数据路径
- 禁止写回安装目录

2. 启动参数固定
- 默认端口
- 日志路径
- 数据目录

3. 异常处理
- 服务启动失败要有明确提示
- 端口被占用要有提示
- 缺少配置文件要自动初始化

4. 首次启动体验
- 自动创建用户数据目录
- 自动生成默认配置文件
- 自动打开主界面

## 打包建议步骤

1. 先完成发布前联测
2. 确认数据目录策略
3. 增加桌面壳入口
4. 在开发机上完成本地打包
5. 在干净机器验证安装与运行
6. 最后补充截图、安装说明、已知限制

## 发布包建议附带文件

- `README`
- `CHANGELOG_V6_0_1.md`
- `RELEASE_CHECKLIST_V6_0_1.md`
- `使用说明`
- `已知限制`

## 当前建议

如果马上进入发布阶段，建议采用：

1. 短期
- 浏览器版 + 启动脚本

2. 中期
- `pywebview` 桌面壳

这是当前风险最低、最符合现有架构的路线。
